# Records

This directory stores reviewed or draft civic-memory records.

The core object is `record`. Records should follow `schemas/record.schema.json`.

Recommended subdirectories for later use:

```txt
records/events/
records/places/
records/organizations/
records/archive-references/
records/photo-references/
records/map-references/
records/timeline/
records/neighbourhoods/
records/cultural-memory/
records/civic-decisions/
records/datasets/
```

Do not place raw harvested dumps here. Use structured records with source IDs, claim IDs, receipt IDs, rights status, public display status, and Ibal review state.
