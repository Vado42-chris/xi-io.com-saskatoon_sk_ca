# xi-io.com, Saskatoon SK CA

This repository preserves structured public records, source references, civic events, historical notes, neighbourhood data, cultural signals, and derived analysis about Saskatoon over time. It is designed to support public-interest research, local journalism, civic dashboards, historical timelines, neighbourhood briefs, and future xi-io publishing workflows.

Governing rule:

> Capture Saskatoon as a living civic memory system, not as a feed of problems.

## Scope

This is a public civic-memory capsule for Saskatoon, Saskatchewan, Canada. It is not a scraper dump, not a police-scanner mirror, and not a finished news site. The first purpose is to preserve source-aware records, receipts, metadata, classifications, and review state so later dashboards and articles can be generated from traceable evidence.

## Architecture

The core object is `record`, not only `event`. A record may describe a place, event, public organization, archive reference, map reference, dataset snapshot, civic decision, historical timeline entry, cultural memory, dashboard metric, or story seed.

The 42-bin taxonomy organizes records into seven rings of six. Safety and emergency response are one ring inside the larger civic-memory system.

Ibal is observer 43. Ibal reviews provenance, rights, place, time, sensitivity, uncertainty, and permitted egress before records become public-facing outputs.

## First milestone

This bootstrap establishes the durable structures only:

1. Mission and policy files
2. City manifest
3. 42-bin taxonomy
4. Local lexicon
5. Source registry scaffold
6. JSON schemas for records, sources, claims, receipts, places, and egress
7. Empty lanes for records, data snapshots, analysis, and Seeds output

No bulk harvest is included in this milestone.

## Preservation stance

Store structured records, source references, receipts, hashes, metadata, summaries, and derived observations. Store full source material only when the rights, licence, and public-interest rules allow it.

For copyrighted news, books, archive holdings, photographs, social posts, and other restricted material, store metadata and claim references, not copied source content.

## Public safety stance

Do not publish exact private addresses, personal identifiers for vulnerable people, live tactical information, private contact details, or unverified rumours. Sensitive records may be preserved internally as generalized, reviewed, or aggregate-only records.
