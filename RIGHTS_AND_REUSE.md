# Rights and Reuse

This repository is public. Every record should be safe for public storage or clearly marked as blocked from public display.

## Default rule

Do not copy source content into this repository unless the rights status allows it.

When rights are unclear, store metadata and a source reference, not the source itself.

## Rights status values

Use these values in records and source entries:

- `open_license`
- `public_domain`
- `permission_granted`
- `metadata_only`
- `fair_dealing_note_only`
- `restricted`
- `unknown_review_required`
- `blocked`

## Copyright-sensitive materials

For copyrighted local news articles, books, photographs, video, audio, archive scans, and social media posts, the repository should preserve:

- Metadata
- Source reference
- Rights notes
- Public-safe summary
- Extracted claims
- Related record links
- Research notes

It should not preserve full copied content unless redistribution is allowed.

## Archive-sensitive materials

Archive holdings may be indexed without copying the holdings. Store collection name, reference identifier, date range, description, rights notes, and research status.

## Reuse posture

The project should prefer open licences for its own original schemas, metadata structures, derived summaries, and public-safe analysis. A final licence can be selected later after review.
