# Security Policy

This repository is public and must not contain secrets, private personal information, or unsafe location detail.

## Do not commit

- API keys or tokens
- Private addresses
- Private phone numbers or email addresses
- Licence plates
- Private resident reports
- Identifying details about children, victims, witnesses, patients, or vulnerable people
- Raw copyrighted material without permission
- Live tactical emergency details

## Report an issue

If unsafe information is found, create a private remediation path where possible. If only public issue tracking is available, do not repeat the sensitive details in the issue. Describe the file path and general class of problem only.

## Automation safety

Any future automation must default to metadata-only or dry-run mode until reviewed. Generated files must pass schema validation and Ibal review before public egress.
